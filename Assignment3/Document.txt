1:
The event loop is Node.js's mechanism for handling asynchronous operations
 It's a single-threaded model that allows node to perform non-blocking I/O operations 
2:
Libuv is a multi-platform support library
is a C library that provides Node.js with its event loop and asynchronous I/O capabilities 
 it handeles file system operations 

3:
 Node.js delegates asynchronous operations to libuv which uses OS-level async APIs or its thread pool
When an async operation completes, Libuv places the callback in the event queue
 The event loop then picks up these callbacks and executes them when the call stack is empty

4:
Call Stack: Executes synchronous JavaScript code (LIFO structure)
Event Queue: Holds completed async operation callbacks (FIFO structure)
Event Loop: Continuously checks if the call stack is empty if empty, moves callbacks from the event queue to the call stack for execution
